# JoJango CMS - Production Deployment Guide

This guide will help you deploy the JoJango CMS application to a VPS using Docker.

## Prerequisites

- VPS with Ubuntu/Debian/CentOS
- Docker and Docker Compose installed
- Domain name pointing to your VPS
- SSL certificate (Let's Encrypt recommended)

## Quick Start

1. **Clone the repository on your VPS:**
   ```bash
   git clone <your-repo-url>
   cd jojangocms3-modelform
   ```

2. **Configure environment variables:**
   ```bash
   cp .env.prod .env
   nano .env  # Edit with your actual values
   ```

3. **Deploy the application:**
   ```bash
   chmod +x deploy.sh
   ./deploy.sh deploy
   ```

## Environment Configuration

Edit the `.env` file with your production values:

```env
# Django Settings
SECRET_KEY=your-super-secret-key-here-change-this
DEBUG=False
ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com

# Database
DB_NAME=jojangocms
DB_USER=jojango_user
DB_PASSWORD=your-secure-database-password

# Email Settings
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password

# Frontend API URL
REACT_APP_API_URL=https://yourdomain.com/api
```

## Deployment Commands

### Full Deployment
```bash
./deploy.sh deploy
```

### Individual Commands
```bash
./deploy.sh build      # Build all services
./deploy.sh up         # Start services
./deploy.sh down       # Stop services
./deploy.sh restart    # Restart services
./deploy.sh logs       # View logs
./deploy.sh migrate    # Run migrations
./deploy.sh static     # Collect static files
./deploy.sh superuser  # Create admin user
```

## Services Overview

- **backend**: Django API server (Daphne ASGI)
- **frontend-admin**: Admin React application
- **frontend-customer**: Customer React application
- **frontend-driver**: Driver React application
- **db**: PostgreSQL database
- **redis**: Redis cache and WebSocket backend
- **nginx**: Reverse proxy and static file server
- **celery**: Background task worker
- **celery-beat**: Scheduled task manager

## URLs After Deployment

- **Admin Panel**: `https://yourdomain.com/admin`
- **API**: `https://yourdomain.com/api`
- **Admin Frontend**: `https://yourdomain.com/admin-frontend`
- **Customer Frontend**: `https://yourdomain.com/customer`
- **Driver Frontend**: `https://yourdomain.com/driver`

## SSL Configuration

1. **Install Certbot:**
   ```bash
   sudo apt update
   sudo apt install certbot python3-certbot-nginx
   ```

2. **Get SSL certificate:**
   ```bash
   sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
   ```

3. **Update nginx.conf** to use SSL paths generated by Certbot.

## Monitoring

- **View logs**: `./deploy.sh logs`
- **Check service status**: `docker-compose -f docker-compose.prod.yml ps`
- **Check resource usage**: `docker stats`

## Backup Strategy

1. **Database backup:**
   ```bash
   docker-compose -f docker-compose.prod.yml exec db pg_dump -U jojango_user jojangocms > backup.sql
   ```

2. **Media files backup:**
   ```bash
   tar -czf media_backup.tar.gz backend/media/
   ```

## Troubleshooting

### Common Issues

1. **Port 80/443 already in use:**
   - Stop other web servers: `sudo systemctl stop apache2 nginx`
   - Or change nginx ports in docker-compose.prod.yml

2. **Database connection failed:**
   - Check DB_PASSWORD in .env file
   - Ensure db service is healthy: `docker-compose -f docker-compose.prod.yml ps`

3. **Static files not loading:**
   - Run: `./deploy.sh static`
   - Check nginx.conf static file paths

### Logs and Debugging

```bash
# View all logs
./deploy.sh logs

# View specific service logs
docker-compose -f docker-compose.prod.yml logs backend
docker-compose -f docker-compose.prod.yml logs nginx

# Access container shell
docker-compose -f docker-compose.prod.yml exec backend bash
```

## Security Considerations

- Change default SECRET_KEY
- Use strong DB_PASSWORD
- Configure firewall (UFW)
- Keep Docker and dependencies updated
- Use SSL/TLS
- Regular backups
- Monitor logs for suspicious activity

## Performance Optimization

- Configure nginx caching
- Set up Redis session storage
- Use CDN for static files
- Monitor resource usage
- Scale services as needed
